Detailed Description

Graphics that renders GCode as a single object.

Unlike all other graphic objects, the GCodeObject acts as container object for a number of segments. The reason, as explained in the based GraphicsObject class, is that having separate buffers for each segment is extremely inefficient. Rendering them all using one buffer dramatically improves performance. This unfortunately increases complexity of the object by a significant amount.
